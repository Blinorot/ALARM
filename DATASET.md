## Dataset Structure

We collect large corpora of audio data with rich metadata to generate prompts and train a model in a self-generation manner. We use only **original** metadata that comes with each dataset without any artificially generated information. The used datasets are presented below with corresponding caption samples. In addition, we use the human subset of [HeySQuAD](https://huggingface.co/datasets/yijingwu/HeySQuAD_human) and [InstructS2S](https://huggingface.co/datasets/ICTNLP/InstructS2S-200K): 0.57M samples, 1.02K hours. For the dataset links and download scripts, refer to [Dataset Creation](#dataset-creation) section.

## Dataset Creation

Due to license restrictions, we cannot redistribute audio files, so you have to download them yourself using the scripts provided. To obtain a dataset, you need to follow three steps.

**Step 1.** Download all the datasets and convert them datasets to `HuggingFace` format. These scripts will do it for you and preprocess the data (resampling to 16kHz, removal of corruptions, etc.):

```bash
cd scripts/alarm/dataset_collection

for file in *.py; do
    if [[ "$file" != "__init__.py" && \
          "$file" != "utils.py" && \
          "$file" != "private_keys.py" ]]; then
        echo "Running $file"
        python3 "$file"
    fi
done
```

By default, the script will download it to `GIT_REPO_ROOT/data/datasets/raw/$DATASET_NAME` paths. This process may take a while and it might be more efficient to run several `.py` scripts in parallel. To reproduce our `VoxCeleb` pairs, use the train list provided in `GIT_REPO_ROOT/assets/voxceleb_train_list.txt`.

**Step 2.** Generate prompts for each of the datasets, except for the speech instructions subset, which already comes with spoken prompts and corresponding text variants with context. To do it from scratch run the following command:

```bash
cd scripts/alarm

python3 create_prompts_offline.py \
    --batch-size 2048 \
    --cuda-devices=$CUDA_VISIBLE_DEVICES
```

It will download [Qwen/Qwen3-30B-A3B-Instruct-2507-FP8](https://huggingface.co/Qwen/Qwen3-30B-A3B-Instruct-2507-FP8) and use it to generate a set of prompts and filter them out for each of the dataset. The prompts will be saved at `GIT_REPO_ROOT/data/datasets/raw_with_prompts/{$DATASET_NAME}_with_prompts`. This process takes some time and it is recommended to run it on several GPUs. Adjust the batch size to satisfy your memory constraints.

Alternatively, you can download the prompts we created from our [ALARM Corpora](https://huggingface.co/datasets/Blinorot/ALARM-Corpora) and match them with your downloaded data using `dataset_index,unique_id_1,unique_id_2` fields. We provide a script for that:

```bash
cd scripts/alarm

# merges data/datasets/raw datasets with prompts and responses from
# https://huggingface.co/datasets/Blinorot/ALARM-Corpora
python3 merge_with_online_alarm_corpora.py --save-name SAVE_NAME
```

The dataset will be saved in `data/datasets/SAVE_NAME`. The saved dataset is not filtered by default (uses `raw` splits). Add `--merge-filtered` to save an already filtered dataset (see [Dataset Preparation](#dataset-preparation) for details regarding filtering). If you merged with our dataset and use [Qwen3-4B-Thinking-2507](https://huggingface.co/Qwen/Qwen3-4B-Thinking-2507) as an RLM, you can skip step 3.

**Step 3.** Generate responses for the datasets. This script will do it for you:

```bash
cd scripts/alarm

python3 generate_responses.py \
    --cuda-devices=$CUDA_VISIBLE_DEVICES \
    --batch-size 2048 \
    --model-name Qwen/Qwen3-4B-Thinking-2507 \
    --no-use-checker \
    --max-thinking-tokens 1536
```

Adjust the batch size to satisfy your memory constraints. You can change `--max-thinking-tokens` to enhance rephrasing or speed up collection. Furthermore, you can change the `--max-tokens` argument that controls the length of the original response. We use `512` by default.

For speech instructions, use (these datasets do not need rephrasing):

```bash
cd scripts/alarm

python3 generate_instruction_responses.py \
    --cuda-devices=$CUDA_VISIBLE_DEVICES \
    --batch-size 2048 \
    --model-name Qwen/Qwen3-4B-Thinking-2507 \
    --dataset-name "heysquad_human"

python3 generate_instruction_responses.py \
    --cuda-devices=$CUDA_VISIBLE_DEVICES \
    --batch-size 2048 \
    --model-name Qwen/Qwen3-4B-Thinking-2507 \
    --dataset-name "instructs2s" \
    --only-context
```

The responses will be saved at `GIT_REPO_ROOT/data/datasets/responses/$MODEL_NAME/$DATASET_NAME_WITH_ARGS`. The `DATASET_NAME_WITH_ARGS` name indicates the dataset name, the model name, the number of tokens used to generate response (`max_tokens`) and whether rephrasing was applied. See `line 1025` in `scripts/alarm/generate_responses.py` for the full `DATASET_NAME_WITH_ARGS` definition.

Alternatively, you can take the responses we got for [Qwen3-4B-Thinking-2507](https://huggingface.co/Qwen/Qwen3-4B-Thinking-2507) and used in the paper by downloading our [ALARM Corpora](https://huggingface.co/datasets/Blinorot/ALARM-Corpora). You will need to match it with your dataset using `dataset_index,unique_id_1,unique_id_2` fields.

**Now the dataset is ready to use**.

## Dataset Preparation

**Step 1.** Original HeySQuAD human dataset has some examples with wrong transcriptions for the spoken instructions. We filter out these examples after generating responses using:

```bash
cd scripts/alarm

python3 filter_dataset.py \
    --dataset-name "heysquad_human" \
    --threshold 0.5 \
    --threshold_type wer
```

It will load [Whisper Large](https://huggingface.co/openai/whisper-large-v3) and run it on audio files. If Whisper transcription has WER higher than 50% with the `question` field (transcription) provided in the original dataset, such an element is removed.

**Step 2.** Before training, we split the corpora into train and validation partitions by running

```bash
cd scripts/alarm

python3 create_train_val_split.py --val-size 0.10 # 10% for val
```

**Step 3.** To avoid any bugs, we double check files for corrupted audio and filter out examples with more than 60 seconds of input:

```bash
cd scripts/alarm

python3 validate_and_filter_indexes.py \
    --max-duration 60.0 \
    --n-jobs 16
```

**Step 4.** We also note that some responses got too short after rephrasing or the rephrasing mechanism got stuck and returned the thinking process of the rephrasing RLM instead of model's response. To fix this, we filter out too short and too long responses by running:

```bash
cd scripts/alarm

python3 validate_and_filter_answers.py \
    --model-name Qwen/Qwen3-4B-Thinking-2507 \
    --no-use-checker \
    --max-thinking-tokens 1536 \
    --tokens-max-limit 612 \
    --tokens-min-limit 100 \
    --n-jobs 16
```

All arguments except for `tokens-min-limit` and `tokens-max-limit` follow those presented for the `generate_responses.py` script. If you modified them, you need to modify them here as well.
